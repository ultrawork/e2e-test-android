# E2E сценарии: Android v35 — Мобильный UI (NotesListScreen)

> **Дата:** 2026-04-01

## Параметры окружения

| Параметр | Значение | Описание |
|----------|----------|----------|
| Платформа | Android Emulator (API 34+) | AVD с Google APIs |
| Приложение | `com.ultrawork.notes` | Package ID |
| Точка входа | `com.ultrawork.notes/.MainActivity` | Запуск через adb |
| testTag списка | `notes_list` | Compose testTag для LazyColumn |
| testTag поиска | `notes_search_field` | Compose testTag для OutlinedTextField |
| testTag счётчика | `notes_counter` | Compose testTag для NotesCounter |
| testTag карточки | `note_card_{id}` | Compose testTag для каждой Card |

## Предварительные условия для всех сценариев

```bash
# Запуск эмулятора
emulator -avd <AVD_NAME> -no-snapshot-load

# Сборка и установка
./gradlew assembleDebug -PAPI_BASE_URL="http://10.0.2.2:4000/api/"
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

## SC-M01: Запуск приложения и отображение главного экрана

**Тип:** Mobile

**Предусловия:** Приложение установлено на эмулятор. Эмулятор запущен.

**Шаги:**
1. Запустить приложение: `adb shell am start -n com.ultrawork.notes/.MainActivity`
2. Дождаться загрузки главного экрана (≤3 сек).
3. Проверить наличие поисковой строки (`notes_search_field`).
4. Проверить наличие счётчика заметок (`notes_counter`).
5. Проверить наличие списка заметок (`notes_list`).
6. Сделать скриншот главного экрана: `adb shell screencap /sdcard/screen_main.png && adb pull /sdcard/screen_main.png`.

**Ожидаемый результат:** Приложение запускается без крашей. Видны: поисковое поле, счётчик с текстом "5 notes" (или аналогичным), список из 5 карточек заметок. Скриншот сохранён.

**Приоритет:** HIGH

**Режим:** manual

## SC-M02: Ввод текста в поисковую строку и фильтрация списка

**Тип:** Mobile

**Предусловия:** Приложение запущено, главный экран отображается, видны все 5 заметок.

**Шаги:**
1. Нажать (tap) на поле поиска (`notes_search_field`).
2. Убедиться, что мобильная клавиатура появилась на экране.
3. Ввести текст `"Shop"` через `input_text`.
4. Проверить, что список обновился и отображается только карточка "Shopping List".
5. Проверить, что счётчик (`notes_counter`) обновился и показывает "1 of 5" (или аналогичный формат filtered/total).
6. Сделать скриншот отфильтрованного списка.

**Ожидаемый результат:** После ввода "Shop" список содержит ровно 1 карточку — "Shopping List". Счётчик показывает filtered/total. Клавиатура остаётся открытой.

**Приоритет:** HIGH

**Режим:** manual

## SC-M03: Очистка поиска через кнопку "X" (Clear button)

**Тип:** Mobile

**Предусловия:** Приложение запущено. В поисковой строке введён запрос (например, "Meet"), список отфильтрован.

**Шаги:**
1. Tap на поле поиска (`notes_search_field`).
2. Ввести текст `"Meet"` через `input_text`.
3. Убедиться, что кнопка очистки (иконка Clear/X, `contentDescription: "Clear search"`) отображается.
4. Tap на кнопку очистки.
5. Проверить, что поле поиска пустое.
6. Проверить, что список отображает все заметки (5 карточек).
7. Проверить, что счётчик показывает только общее количество (без filtered).

**Ожидаемый результат:** После tap на X поисковое поле очищается, список возвращается к полному отображению (5 заметок), счётчик показывает total без filtered-части.

**Приоритет:** HIGH

**Режим:** automated

## SC-M04: Скрытие клавиатуры через ImeAction.Search

**Тип:** Mobile

**Предусловия:** Приложение запущено. Поле поиска активно, клавиатура открыта.

**Шаги:**
1. Tap на поле поиска (`notes_search_field`), дождаться появления клавиатуры.
2. Ввести текст `"Ideas"` через `input_text`.
3. Нажать `ImeAction.Search` (кнопка "Search" / лупа на клавиатуре) через `press_key(ImeAction.Search)`.
4. Проверить, что клавиатура скрылась.
5. Проверить, что список по-прежнему показывает отфильтрованный результат (1 карточка "Ideas").

**Ожидаемый результат:** После нажатия ImeAction.Search клавиатура закрывается. Список остаётся отфильтрованным по запросу "Ideas". Приложение не крашится.

**Приоритет:** MEDIUM

**Режим:** automated
