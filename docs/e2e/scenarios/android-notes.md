# Manual E2E Test Scenarios — Android Notes

## Введение

Документ описывает ручные end-to-end сценарии тестирования для Android-приложения Notes.
Сценарии покрывают функциональность избранных заметок (favorites), фильтрацию, поиск и базовые CRUD-операции.

- **Приложение:** Notes Android
- **Дата:** 2026-03-22
- **Версия документа:** 1.0

## Предусловия

| # | Условие |
|---|---------|
| 1 | Приложение установлено на эмуляторе или устройстве (debug-сборка) |
| 2 | Чистое состояние: данные приложения очищены перед началом тестирования (`adb shell pm clear com.ultrawork.notes`) |
| 3 | Сеть не требуется — favorites хранятся локально (in-memory / Room) |
| 4 | Между сценариями состояние сбрасывается (см. раздел «Сброс состояния») |

## Наблюдаемые UI-элементы

| Элемент | testTag / описание | Назначение |
|---------|--------------------|------------|
| Заголовок заметки | `note_card_{id}` → `Text(titleMedium)` | Идентификация заметки в списке |
| Иконка «Избранное» | Иконка звезды (filled / outlined) на карточке заметки | Состояние избранного |
| Счётчик заметок | `notes_counter` | Отображает общее количество и количество найденных |
| Поле поиска | `notes_search_field` | Ввод текстового запроса |
| Список заметок | `notes_list` | Контейнер списка (LazyColumn) |
| Фильтр «Только избранные» | Toggle / chip для фильтрации избранных | Переключение режима фильтрации |

## Сценарии

### AN-SC-001: Отключение фильтра «Только избранные» восстанавливает полный список

**Цель:** Убедиться, что при выключении фильтра избранных список заметок восстанавливается полностью.

**Шаги:**

| # | Действие | Ожидаемый результат |
|---|----------|---------------------|
| 1 | Создать заметку «Заметка А» | Заметка появляется в списке |
| 2 | Создать заметку «Заметка Б» | В списке 2 заметки, счётчик показывает «2» |
| 3 | Отметить «Заметку А» как избранную (нажать иконку звезды) | Иконка звезды заполнена (filled) для «Заметки А» |
| 4 | Включить фильтр «Только избранные» | В списке отображается только «Заметка А»; счётчик показывает «1 из 2» |
| 5 | Выключить фильтр «Только избранные» | В списке снова 2 заметки: «Заметка А» и «Заметка Б» |
| 6 | Проверить счётчик | Счётчик показывает «2» (общее количество) |

**Ожидаемый результат:** Фильтрация корректна, список полностью восстанавливается при выключении фильтра, счётчик обновляется.

### AN-SC-002: Фильтр избранных совместно с текстовым поиском

**Цель:** Проверить комбинированную работу фильтра избранных и текстового поиска.

**Шаги:**

| # | Действие | Ожидаемый результат |
|---|----------|---------------------|
| 1 | Создать заметку «Купить молоко» | Заметка в списке |
| 2 | Создать заметку «Купить хлеб» | В списке 2 заметки |
| 3 | Создать заметку «Позвонить маме» | В списке 3 заметки, счётчик «3» |
| 4 | Отметить «Купить молоко» как избранную | Звезда заполнена |
| 5 | Отметить «Позвонить маме» как избранную | Звезда заполнена |
| 6 | Включить фильтр «Только избранные» | В списке 2 заметки: «Купить молоко», «Позвонить маме»; счётчик «2 из 3» |
| 7 | Ввести в поле поиска «Купить» | В списке только «Купить молоко»; счётчик «1 из 3» |
| 8 | Очистить поле поиска | В списке снова 2 избранные заметки |
| 9 | Выключить фильтр | В списке все 3 заметки |

**Ожидаемый результат:** Фильтр избранных и текстовый поиск работают совместно, пересечение результатов корректно, счётчик обновляется при каждом изменении.

### AN-SC-003: Удаление избранной заметки при активном фильтре

**Цель:** Проверить, что удаление заметки при активном фильтре избранных корректно обновляет список.

**Шаги:**

| # | Действие | Ожидаемый результат |
|---|----------|---------------------|
| 1 | Создать заметку «Задача 1» | Заметка в списке |
| 2 | Создать заметку «Задача 2» | В списке 2 заметки |
| 3 | Отметить обе заметки как избранные | Обе звезды заполнены |
| 4 | Включить фильтр «Только избранные» | В списке 2 заметки; счётчик «2 из 2» |
| 5 | Удалить «Задачу 1» (свайп или меню) | «Задача 1» исчезает из списка |
| 6 | Проверить список | В списке остаётся «Задача 2»; счётчик «1 из 1» |
| 7 | Выключить фильтр | В списке по-прежнему только «Задача 2» (удалённая не возвращается) |

**Ожидаемый результат:** Удаление корректно обновляет отфильтрованный и полный список, счётчик пересчитывается.

### AN-SC-004: Создание заметки при активном фильтре избранного (smoke)

**Цель:** Убедиться, что создание новой заметки не вызывает побочных эффектов при активном фильтре.

**Шаги:**

| # | Действие | Ожидаемый результат |
|---|----------|---------------------|
| 1 | Создать заметку «Существующая» и отметить как избранную | Заметка в списке, звезда заполнена |
| 2 | Включить фильтр «Только избранные» | В списке 1 заметка |
| 3 | Создать новую заметку «Новая заметка» (не избранная) | Заметка создана, но НЕ отображается в отфильтрованном списке |
| 4 | Выключить фильтр | В списке 2 заметки: «Существующая» и «Новая заметка» |
| 5 | Проверить счётчик | Счётчик показывает «2» |

**Ожидаемый результат:** Новая не-избранная заметка не появляется при активном фильтре, но видна после его отключения.

### AN-SC-005: Редактирование заметки при активном фильтре избранного (smoke)

**Цель:** Убедиться, что редактирование заметки не нарушает фильтрацию.

**Шаги:**

| # | Действие | Ожидаемый результат |
|---|----------|---------------------|
| 1 | Создать заметку «Оригинал» и отметить как избранную | Заметка в списке, звезда заполнена |
| 2 | Создать заметку «Вторая» (не избранная) | В списке 2 заметки |
| 3 | Включить фильтр «Только избранные» | В списке только «Оригинал» |
| 4 | Открыть «Оригинал» и изменить заголовок на «Отредактировано» | Заголовок обновлён |
| 5 | Вернуться к списку | В отфильтрованном списке отображается «Отредактировано» (по-прежнему избранная) |
| 6 | Выключить фильтр | В списке «Отредактировано» и «Вторая» |

**Ожидаемый результат:** Редактирование не сбрасывает статус избранного и не нарушает фильтрацию.

### AN-SC-006: Удаление заметки при неактивном фильтре избранного (smoke)

**Цель:** Убедиться, что удаление заметки без активного фильтра не влияет на статус избранного других заметок.

**Шаги:**

| # | Действие | Ожидаемый результат |
|---|----------|---------------------|
| 1 | Создать заметку «Избранная» и отметить как избранную | Звезда заполнена |
| 2 | Создать заметку «Обычная» (не избранная) | В списке 2 заметки |
| 3 | Удалить «Обычную» (фильтр выключен) | В списке остаётся «Избранная»; счётчик «1» |
| 4 | Включить фильтр «Только избранные» | В списке по-прежнему «Избранная»; счётчик «1 из 1» |
| 5 | Выключить фильтр | Список не изменился — только «Избранная» |

**Ожидаемый результат:** Удаление обычной заметки не влияет на избранные; фильтр работает корректно после удаления.

## Сброс состояния между сценариями

Для обеспечения изоляции тестов между сценариями необходимо сбросить данные приложения:

```bash
# Очистка данных приложения
adb shell pm clear com.ultrawork.notes

# Альтернатива: удаление только базы данных
adb shell run-as com.ultrawork.notes rm -rf databases/
```

После сброса убедитесь, что:
- Список заметок пуст
- Фильтр «Только избранные» выключен
- Поле поиска пустое
- Счётчик показывает «0»

## Замечания

- Favorites хранятся локально (Room / in-memory). Сценарии не описывают взаимодействие с backend endpoint `toggleFavorite`.
- При обнаружении расхождения между документом и реальным поведением приложения — приоритет у реального поведения; документ подлежит обновлению.
